Regra da Cascata

A regra da cascata no CSS funciona como um mecanismo para resolver conflitos de estilo, definindo uma ordem de prioridade para aplicar regras de CSS. Em caso de regras conflitantes para o mesmo elemento, o navegador determina qual deve ser a propriedade a ser aplicada com base em uma hierarquia de origem (como estilos do navegador, do desenvolvedor ou do próprio usuário), importância (!important), especificidade do seletor e, por fim, a ordem de declaração das regras.

Explicação Central

CASCATA, ESPECIFICIDADE e COMBINADORES são conceitos fundamentais no CSS que determinam como os estilos são aplicados aos elementos HTML, especialmente quando há conflitos entre regras. A Cascata define a prioridade de aplicação das regras, levando em conta a origem, importância e a ordem em que as regras são declaradas. A Especificidade é um algoritmo que atribui pontos aos seletores, determinando qual regra tem mais "poder" para ser aplicada quando múltiplas regras se aplicam ao mesmo elemento. Os Combinadores são símbolos que definem relações entre os seletores, permitindo estilos mais direcionados ao combinar seletores de diferentes elementos para especificar relações de descendência, filiação ou irmandade.

Funcionamento de Valores do CSS e Propriedades

As propriedades margin, padding e border definem espaços, sendo padding o espaço interno entre o conteúdo e a borda, margin o espaço externo entre o elemento e os outros. Border é o limite visual do elemento. A notação de shorthand permite definir esses espaços em uma única propriedade, usando de um a quatro valores para margin e padding na ordem de cima, direita, baixo, esquerda (sentido horário). Existem também sufixos como top, bottom, left, right para aplicar valores a lados específicos e sufixos como block e inline que definem o comportamento do elemento, embora margin e padding não os usem diretamente, mas sim as propriedades direcionais.

Especifidade

A especificidade CSS é um sistema de "poder de escolha" que determina qual regra de estilo é aplicada quando há conflitos entre várias declarações para o mesmo elemento, usando uma pontuação baseada no tipo de seletor. Seletores de ID têm maior especificidade que os de classe, que têm mais que os de elemento. A pontuação é calculada em três ou quatro partes (por exemplo, 1,0,0 para um ID, 0,1,0 para uma classe, 0,0,1 para um elemento), onde valores maiores numa parte (como a dos IDs) sobrepõem automaticamente valores menores.

Pseudo-Classes

As pseudo-classes CSS selecionam elementos com base no seu estado, mas first-child, last-child, nth-child e nth-of-type focam na posição do elemento entre os seus irmãos. :first-child e :last-child selecionam o primeiro e o último filho, respetivamente, enquanto :nth-child() e :nth-of-type() oferecem um controlo mais detalhado sobre a posição, com a diferença crucial de que :nth-child() conta todos os irmãos, e :nth-of-type() conta apenas os irmãos do mesmo tipo de elemento.

Combinação

A combinação de seletores no CSS permite definir estilos de forma mais precisa. Quando usamos o espaço, como em .primeiro h1, o estilo será aplicado a todos os elementos "h1" que estiverem dentro de um elemento com a classe .primeiro, independentemente da profundidade. Já a combinação com o sinal de maior >, como em .primeira > h1, seleciona apenas os elementos "h1" que são filhos diretos de .primeira. Por fim, a combinação com vírgula, como em .primeira h1, h1, permite aplicar a mesma regra a mais de um seletor ao mesmo tempo, ou seja, tanto aos "h1" que estão dentro de .primeira quanto a todos os "h1" da página.

Box-Sizing

A propriedade box-sizing: border-box no CSS altera a forma como as dimensões de um elemento são calculadas, fazendo com que o padding e a borda sejam incluídos dentro do width e height definidos, em vez de serem adicionados externamente. Isso significa que um elemento com width de 150px e border-box terá 150px de largura, mesmo que tenha um padding ou borda.

Informações importantes

  • Espaço (descendente): seleciona todos os elementos que estão dentro de outro, em qualquer nível da hierarquia. Exemplo: .menu li afeta todos os "li" que estejam dentro de .menu, mesmo que haja outras tags no meio. É útil quando queremos aplicar estilo a elementos internos sem precisar usar várias classes.
  • Filho direto (>): seleciona apenas os elementos que são filhos imediatos do seletor anterior. Exemplo: .menu > li afeta apenas os "li" que estão logo abaixo de .menu, ignorando os que estão dentro de sublistas. Esse combinador ajuda a manter mais controle quando há estruturas aninhadas.
  • Irmão adjacente (+): seleciona o elemento que vem imediatamente depois de outro. Exemplo: h1 + p aplica estilo apenas ao parágrafo que vem logo depois do "h1".
  • Irmão geral (~): seleciona todos os elementos irmãos que aparecem depois de outro. Exemplo: h1 ~ p afeta todos os parágrafos que aparecem depois do "h1", não apenas o primeiro.
  • Soma (vírgula ,): permite aplicar o mesmo estilo a vários seletores diferentes ao mesmo tempo. Exemplo: h1, h2, h3 aplica a mesma regra a todos esses elementos sem precisar repetir o código.
